System.UInt32 pollingIntervalMs

How long to wait between each I/O loop, in milliseconds
Remarks

Instance state is not thread safe. In other words, when using polling, updating instance state (such as OutputState) both inside and outside of OnStatePolled may create unexpected results. When using polling, it is generally expected you will only make modifications to state inside the OnStatePolled handler in response to input, or outside of the handler in response to external events (for example, game logic). It's also expected that you will only use the DualSense instance passed as an argument to the sender, rather than external references to instance.
